1 因子

其实org-mode本身就自带html生成函数,在你的emacs中c-c c-e就能看到.那么这篇文章的意义在与,c-c c-e中,只有一个网站的基本框架,没有css,图片等细节.如果想要加入,就需要这篇文章帮助了

2 原理

本质上还是利用了org-mode自带的生成函数org-html-pusblish-to-html,只不过我们向其中插入了足够多的辅助性代码.

3 初期准备1

3.1 新建文件

新建文件夹org,里面存放了我们的代码.再新建子文件夹css与img,存放css文件和img文件.结构如下:

org
存放org文件
css
存放css文件
img
存放图形文件

3.2 新建配置.

配置主要分为三项.在你的.emacs中输入 

(require 'ox-publish)
(setq org-publish-project-alist 
  '(("org-notes"
      :base-directory "~/org/"
      :base-extension "org"
      :publishing-directory "~/org/public_html/"
      :recursive t
      :publishing-function org-html-publish-to-html
      :headline-levels 4             ; Just the default for this project.
      :auto-preamble t
      :html-head  "<link rel=\"stylesheet\" type=\"text/css\" href=\"css/stylesheet.css\" />")
     ("org-static"
      :base-directory "~/org/"
      :base-extension "css\\|js\\|png\\|jpg\\|gif\\|pdf\\|mp3\\|ogg\\|swf"
      :publishing-directory "~/org/public_html/"
      :recursive t
      :publishing-function org-publish-attachment
      )
     ("org" :components ("org-notes" "org-static"))))
org-notes
用来推送html文件
org-static
用来推送css文件
org
用来推送整个工程

3.3 如何使用

3.3.1 第一次使用

直接 M-x org-publish-project RET org RET

3.3.2 只想更新html文件

M-x org-publish-project RET org-notes RET

4 须知

如果出现了修改了没有提交上去的情况,说明是org垃圾更新出了问题,这时候就需要在如何使用那一栏里给每一个命令加一个前缀 c-u 就好了,
